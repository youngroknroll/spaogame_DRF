# SPAOGAME MVP 테스트 시나리오 목록

## 공통 정책

* 로그인하지 않은 사용자는 보호된 기능을 사용할 수 없다 [AUTH 필요]
* 잘못된 입력은 실패 사유가 명확히 전달된다
* 존재하지 않는 자원은 조회하거나 조작할 수 없다

---

## 1. 회원 (Users)

### 회원가입

* (완료)사용자는 필수 정보를 입력하면 회원가입할 수 있다
* (완료)이미 가입된 이메일로는 회원가입할 수 없다
* (완료)이메일 형식이 아니면 회원가입할 수 없다
* (완료)비밀번호 규칙을 만족하지 않으면 회원가입할 수 없다
* (완료)회원가입 시 비밀번호는 평문으로 저장되지 않는다

### 로그인

* (완료)사용자는 올바른 이메일과 비밀번호로 로그인할 수 있다
* (완료)로그인에 성공하면 인증 토큰을 발급받는다
* (완료)비밀번호가 틀리면 로그인할 수 없다
* (완료)존재하지 않는 이메일로는 로그인할 수 없다

---

## 2. 상품 (Products)

### 메뉴

* (완료)사용자는 메뉴 목록을 조회할 수 있다
* (완료)관리자는 메뉴를 등록할 수 있다 [ADMIN]

### 카테고리

* (완료)사용자는 특정 메뉴에 속한 카테고리 목록을 조회할 수 있다
* (완료)관리자는 카테고리를 등록할 수 있다 [ADMIN]

### 상품 목록 및 상세

* (완료)사용자는 특정 메뉴와 카테고리에 속한 상품 목록을 조회할 수 있다
* (완료)상품 목록은 페이지 단위로 나누어 조회된다
* (완료)상품 목록은 정렬 기준에 따라 조회할 수 있다
* (완료)사용자는 상품 상세 정보를 조회할 수 있다
* (완료)존재하지 않는 상품은 조회할 수 없다

### 상품 등록

* (완료)관리자는 상품을 등록할 수 있다 [ADMIN]
* (완료)관리자가 아닌 사용자는 상품을 등록할 수 없다

---

## 3. 장바구니 (Cart / Orders)

* (완료)로그인한 사용자는 자신의 장바구니를 조회할 수 있다 [AUTH 필요]
* (완료)장바구니가 비어 있어도 정상적으로 조회된다 [AUTH 필요]
* (완료)로그인한 사용자는 상품을 장바구니에 담을 수 있다 [AUTH 필요]
* (완료)로그인한 사용자는 장바구니 상품의 수량을 변경할 수 있다 [AUTH 필요]
* (완료)로그인한 사용자는 장바구니에서 상품을 제거할 수 있다 [AUTH 필요]
* (완료)다른 사용자의 장바구니는 수정하거나 삭제할 수 없다 [AUTH 필요]

* (완료)로그인한 사용자는 상품에 대한 후기를 작성할 수 있다 [AUTH 필요]

* (완료)댓글 작성자가 아닌 사용자는 댓글을 삭제할 수 없다 [AUTH 필요]

## 5. 문서 대비 추가 누락 항목 (Backlog)

우선순위 표기:
- P1: 문서와의 동작 정합성에 직접 영향
- P2: 확장 기능/부가 정보

### Users 상세 필드

* (완료)[P1]회원가입 시 username, mobile_number, address1, address2, birthday, gender 를 저장할 수 있다
* (미완료)[P2]gender 목록을 조회할 수 있다

### Products 확장 기능

* (완료)[P1]상품 메인 썸네일 목록을 조회할 수 있다
* (완료)[P1]상품 상세에서 색상/사이즈/이미지 정보가 포함된다
* (완료)[P2]상품 상세에 후기/댓글 집계 정보가 포함된다

### Products 상세 도메인

* (완료)[P1]색상(Color)을 등록/조회할 수 있다
* (완료)[P1]사이즈(Size)를 등록/조회할 수 있다
* (완료)[P1]상품 이미지(Image)를 등록/조회할 수 있다
* (완료)[P1]상세 상품(DetailedProduct: color/size/product)을 등록/조회할 수 있다

### Orders/Wishlist

* (미완료)[P2]위시리스트를 추가/조회/삭제할 수 있다 [AUTH 필요]

### Orders/Cart (상세 상품 기반)

* (완료)[P1]장바구니에 상세 상품(color/size 포함)을 담을 수 있다 [AUTH 필요]
* (완료)[P1]장바구니 항목의 상세 상품 정보를 포함해 조회된다 [AUTH 필요]
